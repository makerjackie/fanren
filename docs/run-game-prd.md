在我们的网页当中，加入下面的小游戏（我判断：**这个 idea 是 OK 的，而且适合做成网页里的轻量互动模块。**

但它不要做成“完整跑酷游戏”，而应该做成：

# **韩跑跑：修仙版 Chrome 小恐龙**

一句话：

> 用户扮演韩立，在修仙路上不断躲避 Boss、天劫、妖兽、阵法和因果追杀，跑得越久，境界越高，剧情越离谱。

它的优势是：

1. **梗非常准确**：韩立“韩跑跑”的人设天然适合跑酷。
2. **玩法足够轻**：跳跃、下蹲、闪避、道具即可，不需要复杂地图。
3. **很适合嵌入网页**：可以放在《韩立修仙时间线》某个节点中，当作彩蛋或互动模块。
4. **容易传播**：用户玩完可以截图“我苟到了元婴期 / 我被墨大夫抓了”。
5. **素材可控**：Sprite Sheet 只需要角色跑步、跳跃、受击、Boss 追逐、障碍物几类。

# 核心玩法

类型：

> 轻量 2D 横版无尽跑酷游戏。

参考感觉：

* Chrome 小恐龙
* Flappy Bird 的轻量上手
* Alto’s Adventure 的清爽氛围
* Vampire Survivors 的轻量成长反馈，但不要复杂战斗

用户控制韩立不断向右跑，后方有 Boss / 因果 / 天劫追赶。前方不断出现障碍物和机缘。用户需要跳跃、下蹲、闪避、拾取道具，尽量活得更久。

---

# 核心循环

```text
开始奔跑
→ 躲避障碍
→ 拾取机缘
→ 境界提升
→ 速度变快
→ Boss 更强
→ 失误被抓
→ 获得称号
→ 再来一局
```

这个循环要非常短。

## 敌人追赶系统

后方有一个“追击者”。

第一版不要做真实 AI，只做视觉上的压迫感。

追击者可以根据阶段变化：

| 阶段   | 追击者       |
| ---- | --------- |
| 新手期  | 墨大夫       |
| 黄枫谷期 | 魔道修士      |
| 乱星海期 | 极阴祖师 / 妖兽 |
| 元婴期  | 古魔        |
| 飞升前  | 天劫 / 因果黑影 |

玩法上：

* 玩家撞障碍会减速。
* 减速太多，Boss 距离变近。
* Boss 追上后游戏结束。
* 分数越高，Boss 越离谱。

这比“碰到障碍直接死”更适合韩跑跑，因为韩立的核心不是无伤，而是“苟住”。


每次突破时：

* 画面闪光
* 速度略微提升

比如：

```text

突破结丹期：跑路速度大幅提升。
```

这个是“反馈感”的核心。

# 技术建议

你说不要重引擎，这个判断对。

## 不建议

不建议第一版用：

* Unity WebGL：太重。
* Unreal：完全没必要。
* Phaser：能用，但对这个项目可能略重。
* PixiJS：如果你要大量粒子和精灵，可以用，但第一版也不是必须。

## 推荐方案

### 方案 A：React + Canvas

适合你当前网页项目。

技术：

* React 负责页面 UI
* Canvas 负责游戏渲染
* requestAnimationFrame 做游戏循环
* 简单碰撞检测
* Sprite Sheet 动画
* Zustand / useReducer 管状态

优点：

* 很轻
* 易嵌入
* 和网页融合好
* 不需要游戏引擎
* Codex 好改

推荐度：最高。

---

### 方案 B：Kaboom.js

Kaboom.js 是一个轻量 2D 游戏库，做小网页游戏很快。

优点：

* 比 Phaser 更轻
* 语法简单
* 适合小型跑酷
* 快速做原型

缺点：

* 和 React UI 融合需要稍微处理
* 后续如果要非常定制，可能还是自己写 Canvas 更稳

推荐度：中高。

---

### 方案 C：PixiJS

如果你非常重视丝滑视觉、粒子、光效、精灵动画，可以用 PixiJS。

优点：

* 性能好
* 适合 2D 精灵
* 视觉表现更强

缺点：

* 比 Canvas 手写复杂一点
* 对 MVP 来说可能略重

推荐度：中。

# 需求文档草案

```md
# 韩跑跑 MVP 需求文档

## 1. 项目定位

《韩跑跑》是嵌入在《凡人修仙时间线》网站中的轻量互动小游戏。用户扮演韩立，在修仙路上不断奔跑、躲避障碍、拾取机缘、突破境界，并尽可能避免被后方 Boss 追上。

游戏整体风格轻量、清亮、有梗，强调“韩立式苟道生存”，不是复杂动作游戏，也不是重度 RPG。

## 2. 核心体验

用户进入游戏后，韩立会自动向前奔跑。玩家通过跳跃和下蹲躲避障碍，拾取小绿瓶、符箓、丹药等道具来延长生存时间。随着分数提高，韩立会从凡人逐步突破到炼气、筑基、结丹、元婴等境界。游戏失败后展示本局境界、距离、死因和搞笑称号。

核心体验目标：

- 10 秒内理解玩法
- 30 秒内获得第一次笑点
- 1 分钟内出现境界突破反馈
- 3 分钟内愿意再来一局

## 3. 游戏类型

2D 横版无尽跑酷小游戏。

参考方向：

- Chrome 小恐龙的简单操作
- Alto’s Adventure 的清爽氛围
- 修仙题材的成长反馈和梗文化

## 4. 核心玩法循环

开始奔跑 → 躲避障碍 → 拾取道具 → 获得分数 → 境界突破 → 速度提升 → 障碍变难 → 被追上或失误 → 结算称号 → 再来一局。

## 5. 操作方式

### 桌面端

- Space / ↑：跳跃
- ↓：下蹲
- P：暂停

### 移动端

- 点击：跳跃
- 下滑：下蹲

MVP 第一版只实现跳跃和下蹲。

## 6. 玩家角色

玩家角色为韩立的轻量化形象。

需要动画状态：

- idle
- run
- jump
- crouch
- hit
- caught

## 7. 追击系统

游戏左侧后方存在一个追击者，代表剧情中的 Boss、天劫或因果。

规则：

- 玩家正常奔跑时，与 Boss 保持安全距离。
- 撞到障碍会短暂减速，Boss 距离拉近。
- 拾取丹药或飞剑可拉开距离。
- Boss 距离归零时，游戏结束。
- 随着分数提升，Boss 形象和压迫感逐步增强。

MVP 中 Boss 不需要复杂 AI，只需要根据玩家失误和游戏进度改变距离。

## 8. 障碍物

MVP 障碍分为两类：

### 地面障碍

需要跳跃躲避。

示例：

- 石块
- 阵法裂缝
- 魔焰

### 空中障碍

需要下蹲躲避。

示例：

- 飞剑
- 妖禽
- 雷弧

障碍物会从右向左移动，速度随游戏进度逐渐提升。

## 9. 道具

MVP 实现 3 种道具：

### 小绿瓶

效果：短暂无敌，持续 3 秒。

### 符箓

效果：抵挡一次障碍伤害。

### 丹药

效果：恢复一定 Boss 距离。

## 10. 境界系统

根据分数自动突破境界。

境界阶段：

1. 凡人
2. 炼气
3. 筑基
4. 结丹
5. 元婴
6. 化神

每次突破时：

- 显示突破动画
- 播放短文案
- 背景氛围轻微变化
- 游戏速度略微提升

示例文案：

- 炼气：终于踏入修仙门槛。
- 筑基：苟道初成。
- 结丹：跑路速度大幅提升。
- 元婴：韩老魔上线。

## 11. 分数与结算

游戏记录：

- 跑路距离
- 存活时间
- 当前境界
- 收集灵石数量
- 死因
- 称号

结算页示例：

跑路距离：3280 里  
收集灵石：42  
死因：被极阴祖师追上  
称号：谨慎但不够快

## 12. 死因文案

根据失败原因显示不同文案：

- 被 Boss 追上：跑慢一步，因果加身。
- 撞到飞剑：谨慎了一生，还是没躲过这一剑。
- 被雷劈中：道友，雷劫不讲武德。
- 撞到阵法：你误入了不该进的阵。
- 高分失败：韩老魔也有翻车的时候。

## 13. 视觉风格

整体风格清亮、轻量、古风、带一点幽默感。

关键词：

- 清亮
- 古风
```

# 另一个备选 idea

如果你觉得跑酷太常见，还有一个更适合韩立的轻量玩法：

## **韩立苟道模拟器**

用户每回合遇到事件，选择：

* 跑
* 苟

* 寿命
* 境界
* 资源
* 仇恨
* 因果

> **韩立一直跑，玩家负责帮他苟住。**
